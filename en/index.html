Getting Started

Build your own data-driven REST API in no time

  • Highly adaptable REST API
  • Super fast in-memory database
  • No setup or configuration required (ready to use)
  • High data security and data protection
  • Perfect for quick prototyping and smaller projects

Getting started with the public server

First, let's create a simple resource using HTTP PUT. Hit the "Run" button and see what happens.

PUT /review123456/beers/pale-ale HTTP/1.1 
Host: public.demo.restful.services
Content-Type: application/json

{
  "type": "Pale Ale",
  "flavor": 9
}


        

      

      
The server response should inform you about successfully adding a resource. You just saved a JSON object
        to a freely definable path!

      

        
The editors on this website use HTTP syntax in a very basic form. The code is parsed in your browser and sent
          directly via your browser's Fetch API.

        
So you can change the examples as you like. However, in order to be able to follow the instructions here
          well, you should not change them too much for the beginning.

      

      
You can get the resource you just created with HTTP GET:

      

        

          

            
GET /review123456/beers/pale-ale HTTP/1.1 
Host: public.demo.restful.services


            
          

        

        

          
          
        

        

          

        

      

      
Another way to add a resouce is HTTP POST. In contrast to HTTP PUT, a new, unique
        resource ID is automatically created for you:

      

        

          

            
POST /review123456/beers HTTP/1.1 
Host: public.demo.restful.services
Content-Type: application/json

{
  "type": "Imperial Stout",
  "flavor": 6
}


            
          

        

        

          
          
        

        

          

        

      

      

        
You can add any type of resource. Even binary resources such as images or proprietary
          formats are supported.

      

      
With HTTP GET, not only single resources can be requested, but also a collection of resources.
        Collections give you an overview of all direct child resources:

      

        

          

            
GET /review123456/beers HTTP/1.1 
Host: public.demo.restful.services


            
          

        

        

          
          
        

        

          

        

      

      
You can use various URL queries to get different responses and, for
        example, include the actual
        data or do
        some sorting or filtering. And all without having to configure the REST API beforehand. Pretty cool right?

      
RESTful Services also provides additional features like SSE (server-sent events) or webhooks to react
        immediately on a change in your database.
